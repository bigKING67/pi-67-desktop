# Contributing

1. Read `AGENTS.md`, `PRODUCT.md`, `DESIGN.md`, and the active OpenSpec change.
2. Keep changes within the documented layer dependency direction.
3. Add or update tests for protocol, policy, and projection behavior.
4. Restore packages in locked mode and keep warnings-as-errors green.
5. Never commit credentials, user sessions, logs, databases, screenshots,
   traces, build output, or installer output.
6. Report Windows build, runtime, accessibility, and performance evidence
   separately; one does not imply the others.

External publication, signing, releases, and pushes are separate authorized
actions and are not implied by a local implementation request.
